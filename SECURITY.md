# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this project, please report it responsibly.

### What to Report
- Security vulnerabilities in scripts or automation tools
- Malicious content that may have been injected via contribution
- Broken links leading to malicious domains

### How to Report
1. **Do NOT** create a public GitHub issue
2. Use [GitHub Security Advisories](https://github.com/Ak-cybe/awesome-offensive-security-skills/security/advisories/new)
3. Include: description, reproduction steps, and potential impact

### Response Timeline
- **Acknowledgment**: Within 48 hours
- **Investigation**: Within 7 days
- **Resolution**: Within 30 days

## Scope

This project contains **educational offensive security content**. The skills themselves describe attack techniques — this is intentional and expected. Security reports about skill content being "offensive" are out of scope.

In-scope issues include:
- Malicious code in scripts/ directories that performs unintended actions
- Supply chain attacks via dependencies
- XSS or injection in the eval-viewer HTML
- Privacy violations in data collection

All skills assume authorized use. See [LICENSE](./LICENSE) and individual skill prerequisites.
